# 発表資料用解説 - 索引＆使用ガイド

発表資料を作成するためのわかりやすい解説ドキュメント集です。  
すべて **初心者向け** に書かれているので、非技術者にも説明できます。

---

## 📚 ドキュメント一覧

### 1️⃣ [プロジェクト概要](PRESENTATION_01_OVERVIEW.md)

**対象**: 「このシステムって何？」を説明する時

**内容**:

- システムの目的・機能
- 3種類のユーザー (ゲスト・ユーザー・管理者)
- 従来の方法との比較
- 実際の使用例

**分量**: 中程度  
**活用例**:

- 冒頭で「全体像」を説明
- ステークホルダー向け

---

### 2️⃣ [ユーザーと管理者の動き](PRESENTATION_02_USER_FLOW.md)

**対象**: 「どうやって使うの？」を説明する時

**内容**:

- ユーザー（認証会員）の登録～試験実施の流れ
- 管理者の イベント作成～成績管理の流れ
- ステップバイステップの図解

**分量**: 大きめ (詳細なフロー)  
**活用例**:

- 「ユーザーは何ができるのか」を説明
- 「管理者は何をするのか」を説明
- デモンストレーション時の補足

---

### 3️⃣ [ゲストユーザーの試験フロー](PRESENTATION_03_GUEST_FLOW.md)

**対象**: 「コード入力だけで受験？」を説明する時

**内容**:

- セッションコード入力方法
- ゲスト情報入力
- ゲスト試験実施の流れ
- ユーザーとゲストの違い
- 実際のシーン (企業説明会・講義)

**分量**: 中程度  
**活用例**:

- 企業説明会での説明
- 学生向けの簡潔な説明

---

### 4️⃣ [データベースの役割](PRESENTATION_04_DATABASE.md)

**対象**: 「どこにデータが保存されるの？」を説明する時

**内容**:

- データベースとは (簡単説明)
- 保存されるデータの種類
- テーブル間の関係
- データベースなかったら?
- セキュリティ対策

**分量**: 中程度  
**活用例**:

- 技術的な質問への回答
- データ管理の説明
- セキュリティ説明の補足

---

### 5️⃣ [セキュリティ機能](PRESENTATION_05_SECURITY.md)

**対象**: 「安全なの？」を説明する時

**内容**:

- ログイン認証
- パスワード暗号化
- CSRF 対策
- セッション管理
- 不正検知システム
- 多層防御の考え方

**分量**: 大きめ (詳細な説明)  
**活用例**:

- 「このシステムは安全です」という説明
- 「不正検知機能があります」という説明
- 保護者・学校向けの信頼構築

---

### 6️⃣ [技術スタック](PRESENTATION_06_TECHNOLOGY.md)

**対象**: 「どんな技術を使ってるの？」を説明する時

**内容**:

- フロントエンド技術 (Vue.js, Tailwind CSS)
- バックエンド技術 (Laravel, PHP)
- データベース (MySQL)
- 実行環境 (Docker, Sail)

- システムの強み

**分量**: 大きめ (技術詳細)  
**活用例**:

- 技術者向けの説明
- 「なぜこれ選んだ？」への回答
- 拡張性・セキュリティの説明

## 💡 スライド作成のコツ

### 1️⃣ 情報を絞る

```
❌ 「このファイルすべてを入れよう」
   → スライドがごちゃごちゃ
   → 観客が疲れる

✅ 「最重要な3-5ポイントだけ入れよう」
   → スライドが見やすい
   → メッセージが伝わる
```

### 2️⃣ 図解を活用

```
テキストだけ → 退屈
   ↓
図・表・フロー → わかりやすい
   ↓
図 + 簡潔なテキスト → 最適
```

### 3️⃣ 具体例を交える

```
抽象的: 「ユーザーが試験を受けます」
   ↓ (わかりにくい)

具体的: 「田中太郎さんがログイン
       → 試験開始 → 90問解く
       → 75点の結果が表示される」
   ↓ (わかりやすい)
```

### 4️⃣ 発表者向けの補足資料を用意

```
スライド: 「ユーザーと管理者がいます」

補足資料 (このドキュメント):
「ユーザーは以下ができます...
 管理者は以下をします...」
```

## 🎤 発表時の心がけ

### 1️⃣ 聴き手に合わせて説明のレベルを調整

```
技術者向け:
「REST API, OAuth 2.0, ...」

非技術者向け:
「ユーザーがクリック → サーバーが処理」
```

### 2️⃣ 専門用語を避ける

```
❌ 「CSRF トークン」
✅ 「悪いサイトからの攻撃から守る仕組み」
```

### 3️⃣ 相互作用を大切に

```
「何か質問ありますか？」
「ここまでで理解できていますか？」

質問に答えるときは、相手の知識レベルを考慮
```

## 📱 各ドキュメントの「5行サマリー」

### [1] プロジェクト概要

> オンラインで適性試験を受けるシステム  
> 3種類のユーザー (ゲスト・ユーザー・管理者)  
> 試験は自動採点、すぐに結果表示  
> どこからでも、いつでも受験可能  
> 不正検知機能付き

### [2] ユーザー・管理者の動き

> ユーザーは登録 → ログイン → 試験実施 → 結果確認  
> 管理者はイベント作成 → 問題設定 → 成績確認  
> ステップバイステップで詳細説明  
> 各ステップの画面イメージを図解  
> 不正検知ログ確認まで

### [3] ゲストフロー

> 登録なしでコード入力のみでアクセス  
> セッションコード → ゲスト情報 → 試験実施  
> その場で結果表示・メール送信  
> ユーザーとの主な違いを比較表示  
> 企業説明会・講義での利用シーン

### [4] データベース

> システムの情報を保存する倉庫  
> ユーザー・問題・成績・不正ログを管理  
> テーブル構造とリレーションを図解  
> パスワード暗号化などセキュリティ対策  
> 初心者向けの簡潔説明

### [5] セキュリティ

> ログイン認証 / パスワード暗号化 / CSRF対策  
> セッション管理 / 不正検知システム  
> 5つの多層防御で安全性を確保  
> 不正検知ログの対応フロー  
> ユーザーがすべきことも記載

### [6] 技術スタック

> Vue.js (画面) / Laravel (処理) / MySQL (保存)  
> Docker で環境統一  
> 各技術の役割と選定理由  
> スケーラビリティ・セキュリティの強み  
> 開発・実行の流れ
