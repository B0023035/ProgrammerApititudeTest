# 発表資料用解説 - 3. ゲストユーザーの試験フロー

## 🌐 ゲストユーザーとは

```
❌ 登録不要
❌ ログイン不要
✅ コードを入力するだけ
✅ 即座に試験開始可能
```

**使用シーン**:

- 企業説明会で「今すぐ試験を受けてください」
- 大学の講義で「このコード入れてみてください」
- とにかく手軽に試験を受けたい人

---

## 📋 ゲスト試験の流れ

### ステップ1: セッションコード入力

```
サイトのトップページを開く
   ↓
「セッションコードを持っていますか？」
   ↓
コード入力ボックス
   ├─ 例: EXAM202401
   ├─ 例: TEST12345
   └─ 例: COMPANY2024
   ↓
コードを入力
   ↓
「確認」ボタンをクリック
   ↓
┌─────────────────────────────┐
│ サーバーが確認中...          │
│                             │
│ ✓ このコード、存在する？     │
│ ✓ 試験、今実施可能？        │
│ ✓ 開始時刻は過ぎた？        │
│ ✓ 終了時刻内？             │
└─────────────────────────────┘
   ↓
✅ OK → ゲスト情報入力画面へ
❌ NG → 「コードが無効です」エラー表示
```

---

### ステップ2: ゲスト情報入力

```
ゲスト情報入力フォーム
   ↓
【必須項目】
   ├─ 名前（表示用）: 「田中太郎」
   └─ メール（受取用）: 「tanaka@example.com」
   ↓
【オプション項目】
   ├─ 学年（大学3年、高校2年など）
   └─ 企業名（受けている企業の説明会など）
   ↓
「試験を開始」ボタンをクリック
   ↓
✅ 情報保存完了
```

**重要な違い**: ゲストの情報はパスワード不要。
メールは結果送信用だけで、パスワードはありません。

---

### ステップ3: 試験開始

```
試験開始画面
   ├─ 「注意事項をお読みください」
   │
   ├─ ⏱️ 制限時間: 120分
   │  (タイマーがカウントダウン)
   │
   ├─ 📝 問題について
   │  ├─ 全3パート (Part1, 2, 3)
   │  ├─ 各パート30問
   │  └─ 計90問
   │
   ├─ ⚠️ 重要なルール
   │  ├─ 「一度見た問題には戻れません」
   │  ├─ 「解答後は変更できません」
   │  ├─ 「時間が来たら自動終了」
   │  └─ 「ブラウザを閉じると無効」
   │
   └─ 「試験を始める」ボタン
   ↓
ボタンをクリック
```

## 🔒 ゲスト試験の安全性

### 不正監視システム

ゲストの試験中も、システムが不正を監視しています：

```
✓ 別タブに切り替えたか
  → 検知: ⚠️ 「ウィンドウフォーカスが失われました」

✓ 右クリックして情報をコピーしようとしたか
  → 検知: ⚠️ 「コピーは禁止されています」

✓ キーボードショートカット (Ctrl+C など) を使ったか
  → 検知: ⚠️ 「このアクション は許可されていません」
```

これらはすべて **記録** されて、管理者が確認できます。

## ⚡ ユーザーとゲストの主な違い

```
┌─────────────────┬──────────────┬──────────────────┐
│ 項目              │ ゲスト       │ ユーザー（登録）  │
├─────────────────┼──────────────┼──────────────────┤
│ 登録              │ 不要         │ 必須              │
│ パスワード        │ なし         │ あり              │
│ ログイン          │ 不要         │ 毎回必要          │
│ アクセス方法      │ コード       │ メール + パス     │
├─────────────────┼──────────────┼──────────────────┤
│ 試験受験          │ ✓可          │ ✓可              │
│ 練習問題          │ ✗不可        │ ✓可              │
│ 過去成績確認      │ ✗不可        │ ✓可              │
│ 成績保存          │ その場だけ   │ 永続保存          │
│ メール結果受取    │ ✓可          │ ✓可              │
└─────────────────┴──────────────┴──────────────────┘
```

## 💡 典型的なゲスト試験シーン

### シーン: 企業説明会

```
【開始前】
企業HR: 「これから適性診断をしますので、
         このコード『COMPANY2024』を入力してください」

全員: スマートフォン/PCでコード入力

【試験中】
受験者: コード入力 → 名前入力 → 試験開始 → 90問解く

システム: 不正を監視

【試験後】
受験者: すぐに結果表示 → 印刷またはメール受取

企業: 全員の成績をすぐに確認 → 後日面接の参考に
```

### シーン: 大学の講義

```
教授: 「プログラミング能力の確認です。
      このコード『UNIV20240120』を使ってください」

学生: 全員がコード入力 → 受験

教授: 答え合わせの時間短縮
      (従来は手で採点) → (今は自動採点)

学生: その場で成績確認 → 自分の弱点を把握
```

## ✨ まとめ

| ポイント     | 説明                        |
| ------------ | --------------------------- |
| **何か**     | 登録なしで受験できる試験    |
| **アクセス** | コード入力するだけ          |
| **入力情報** | 名前とメール（簡潔）        |
| **試験内容** | Part 1, 2, 3 (計90問)       |
| **結果**     | その場で表示 + メール送信可 |
| **保存期間** | 通常はその場限り            |
| **安全性**   | 不正監視あり                |
| **メリット** | 手軽、登録不要、即結果      |
