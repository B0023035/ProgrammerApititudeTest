# 発表資料用解説 - 1. プロジェクト概要（初心者向け）

## 🎯 このシステムとは？

**「プログラマー適性検査システム」** = オンラインで適性試験を受けるシステム

### 簡単に言うと...

```
📱 ブラウザで → 🧪 試験を受ける → 📊 結果を見る

オンラインで、好きな時間に、好きな場所から
プログラマーの適性を調べるテストが受けられる
```

---

## 👥 3種類のユーザー

このシステムには **3種類の人** が使います：

### 1️⃣ **ゲスト（一般来訪者）**

- 登録なしで試験に参加できる
- 「コード」を入力するだけ
- 名前とメールだけ教えればOK
- 試験を受けて、その場で結果を見る

**例えば：** 企業の説明会で「今からみんなで適性検査を受けてみてください。このコードを入力してください」

```
流れ: コード入力 → 名前入力 → 試験開始 → 結果表示
```

---

### 2️⃣ **ユーザー（登録会員）**

- メールアドレスでアカウント登録
- ログインして試験に参加
- 過去の試験結果を保存できる
- 練習問題が見られる

**例えば：** 大学生が「このサイトに登録して、何回も試験に挑戦する」

```
流れ: 登録 → ログイン → 試験選択 → 試験開始 → 結果保存 → 過去結果確認
```

---

### 3️⃣ **管理者（試験を作る側）**

- イベント（試験）を作成する
- 問題と答えを登録する
- みんなの成績を確認する
- ズルをしていないか監視する

**例えば：** 大学の教授が「1月20日の適性検査を作成して、成績を確認する」

```
流れ: イベント作成 → 問題設定 → 試験実施 → 成績確認
```

## ⚡ なぜこのシステムが必要？

### 従来の方法

```
❌ 紙のテストを印刷
❌ 試験会場を用意
❌ 手で採点
❌ 手で成績集計
❌ 時間と手間がかかる
```

### このシステム

```
✅ ブラウザでどこからでも受験可能
✅ 自動採点
✅ 自動成績集計
✅ すぐに結果表示
✅ 時間と手間を削減
```

---

## 💡 わかりやすい比較

| 項目         | 紙のテスト        | このシステム |
| ------------ | ----------------- | ------------ |
| どこで受ける | 決まった場所      | どこでもOK   |
| いつ受ける   | 決まった日時      | 柔軟に選べる |
| 答え合わせ   | 後日              | すぐに表示   |
| 結果管理     | ファイル/フォルダ | 自動保存     |
| 手間         | 多い              | 少ない       |

## 🎓 実際の使用例

### シーン1: 大学の講義

```
教授: 「来週の講義で適性検査をします。このコードを使ってください」
学生: ブラウザでコード入力 → 受験 → その場で結果確認
教授: 全員の成績をすぐに集計可能
```

### シーン2: 企業説明会

```
企業: 「適性検査を通じて、みなさんがプログラマーに向いてるか試してみましょう」
参加者: 会社のブラウザでコード入力 → 受験 → 結果を持ち帰り
企業: 興味のある学生を把握可能
```

### シーン3: 個人でのスキルチェック

```
大学生: 「プログラミングの適性を確認したい」
      → サイトに登録 → 何回でも受験可能
      → 過去の成績と比較して成長を実感
```

## 🔐 安全性の特徴

### 不正検知機能

システムは **ズルをしていないか** 自動で監視します：

- ✓ 他のタブに切り替えたか
- ✓ 答えをコピーしようとしたか
- ✓ キーボードを操作していないか

不正が検知されたら、管理者が確認して対応します。

## 📱 技術的な特徴（参考）

このシステムは最新のWeb技術で作られています：

- **Laravel**: サーバー側 (PHP)
- **Vue.js**: クライアント側 (JavaScript)
- **MySQL**: データベース
- **Docker**: システム実行環境

但し、発表では「Web技術を使った最新のシステムです」程度で十分です。
