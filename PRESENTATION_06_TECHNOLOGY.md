# 発表資料用解説 - 6. 技術スタック（初心者向け）

## 🛠️ 「技術スタック」とは

```
【簡単な説明】
「このシステムを作った技術・道具」

【例え】
建築物でいう「どんな素材を使ったか」
  ├─ 壁: コンクリート
  ├─ 屋根: 瓦
  ├─ 柱: 鉄骨
  └─ 窓: ガラス
```

---

## 📦 このシステムで使っている技術

### 1️⃣ **フロントエンド** (ユーザーが見る部分)

#### Vue.js (ビュー・ジェイエス)

```
【何か】
  ユーザーのブラウザで動く
  JavaScriptのフレームワーク

【役割】
  ├─ 画面デザイン
  ├─ ユーザー操作 (クリック、入力など)
  ├─ データ表示
  └─ 画面の動きを作る

【例】
  ユーザーが「次の問題」をクリック
   ↓
  Vue.js が即座に反応
   ↓
  新しい問題が表示される (スムーズに)
```

#### Tailwind CSS (タイルウィンド・シーエスエス)

```
【何か】
  デザイン・見栄えを整える技術

【役割】
  ├─ 画面のレイアウト
  ├─ 色・フォント・間隔
  ├─ ボタンのデザイン
  └─ 美しく見やすくする

【例】
  「大きなボタン」 → 重要
  「小さいテキスト」 → 補足情報
```

#### Inertia.js (イナーシャ・ジェイエス)

```
【何か】
  フロントエンドとバックエンドを繋ぐ技術

【役割】
  「Vue.js で描かれたページ」と
  「サーバーが送る情報」を結びつける

【メリット】
  ├─ ページ遷移が速い
  ├─ 毎回全ページを読み込まない
  └─ スムーズなユーザー体験
```

---

### 2️⃣ **バックエンド** (サーバー側の処理)

#### Laravel (ララベル)

```
【何か】
  PHP というプログラミング言語のフレームワーク
  (フレームワーク = 枠組み・ひな形)

【役割】
  ├─ リクエスト受け取り
  ├─ ビジネスロジック処理
  │  (認証、採点、不正検知など)
  ├─ データベース操作
  └─ レスポンス返却

【例】
  ユーザーがログイン
   ↓ リクエスト送信
  Laravel が処理
   ├─ メール存在確認
   ├─ パスワード確認
   ├─ セッション作成
   └─ OK/NG 返す

【利点】
  ├─ 大規模開発に適している
  ├─ セキュリティ機能が豊富
  ├─ データベース操作が簡単
  └─ 多くの企業が使っている
```

#### PHP (ピーエイチピー)

```
【何か】
  サーバー側で動くプログラミング言語

【役割】
  サーバーで処理を実行
  ├─ 計算
  ├─ ファイル操作
  ├─ データベース操作
  └─ メール送信

【特徴】
  ├─ Web開発に最適
  ├─ 簡単に学べる
  └─ 多くのサーバーで対応
```

---

### 3️⃣ **データベース**

#### MySQL (マイエスキューエル)

```
【何か】
  大量のデータを整理・保存するシステム

【役割】
  ├─ ユーザー情報を保存
  ├─ 問題を保存
  ├─ 解答を保存
  ├─ 成績を保存
  └─ 不正ログを保存

【特徴】
  ├─ 広く使われている (業界標準)
  ├─ 信頼性が高い
  ├─ 高速
  └─ 無料で使える

【例】
  ユーザー登録
   ↓
  MySQL に保存
   ↓
  何年後でも参照可能
```

#### Eloquent ORM

```
【何か】
  データベースを簡単に操作するツール

【役割】
  Laravel で MySQL を簡単に使う

【例】
  通常: SQL文を書く (複雑)
  Eloquent: シンプルなコードで可能
```

---

### 4️⃣ **実行環境・デプロイメント**

#### Docker (ドッカー)

```
【何か】
  システムを「容器」に詰め込む技術

【例え】
  「自分のパソコンで動いたけど、
   別のパソコンでは動かない」
  という問題を解決する

【役割】
  ├─ 開発環境と本番環境を統一
  ├─ どのサーバーでも同じように動く
  ├─ インストール手間を削減
  └─ セットアップが簡単

【メリット】
  ├─ 誰が実行しても同じ結果
  ├─ テスト環境のセットアップが簡単
  └─ デプロイが楽
```

#### Laravel Sail

```
【何か】
  Docker を Laravel 向けに簡潔にしたツール

【役割】
  ├─ Docker 環境をワンコマンドで起動
  ├─ MySQL・Redis など必要なツール自動設定
  └─ 開発者は開発に集中できる

【利点】
  ├─ 複雑な Docker 設定が不要
  ├─ 開発が速い
  └─ トラブルが少ない
```

## 🎓 なぜこれらの技術を選んだのか？

### 1️⃣ Vue.js + Tailwind CSS

```
理由:
  ✓ 最新で標準的な技術
  ✓ 学習リソースが豊富
  ✓ パフォーマンスが良い
  ✓ 小規模から大規模まで対応
```

### 2️⃣ Laravel

```
理由:
  ✓ セキュリティ機能が充実
  ✓ データベース操作が簡単
  ✓ 大企業でも使用
  ✓ 開発速度が速い
```

### 3️⃣ MySQL

```
理由:
  ✓ 業界標準
  ✓ 無料
  ✓ 信頼性が高い
  ✓ スケーラビリティ (拡張可能)
```

### 4️⃣ Docker

```
理由:
  ✓ 環境の統一
  ✓ デプロイが楽
  ✓ チーム開発に最適
```

## 💻 開発・実行の流れ

```
【開発フェーズ】

1️⃣ コード作成
   ├─ Vue.js コンポーネント
   ├─ Laravel コントローラー
   └─ SQL クエリ

2️⃣ Docker で実行
   $ docker-compose up
   → 自動でセットアップ・実行

3️⃣ ブラウザでテスト
   http://localhost
   → 確認

4️⃣ バグ修正
   ↓ (2️⃣-3️⃣ を繰り返す)

【本番デプロイ】

5️⃣ コード整理
   ├─ テスト実行
   ├─ セキュリティチェック
   └─ パフォーマンス確認

6️⃣ 本番サーバーにアップロード
   Docker で統一環境構築

7️⃣ 公開
   ユーザーがアクセス可能
```

### スケーラビリティ（拡張性）

```
【今】
  ├─ ユーザー: 100人
  ├─ 問題: 90問
  └─ 成績記録: 100件

【将来】
  ├─ ユーザー: 10,000人 (100倍)
  ├─ 問題: 1000問
  └─ 成績記録: 100,000件

技術選択で対応可能 ✓
  ├─ MySQL は大容量に対応
  ├─ Laravel はチューニング可能
  └─ Docker で環境拡張可能
```

```
Laravel の標準機能:
  ✓ CSRF 保護 (自動)

  ✓ 暗号化機能
  ✓ 認証フレームワーク
  ✓ レート制限
```

  ✓ 仮想DOM で高速描画
  ✓ 差分更新で無駄削減

Laravel:

  ✓ クエリ最適化ツール
