## 1. プレゼンテーション向けQ&A

### 開発・設計に関する質問

#### Q: なぜこの技術スタック（Laravel + Vue.js）を選択したのですか？

**A:** 以下の理由から選択しました：

- **Laravel**: PHPの中で最も人気のあるフレームワークであり、認証・セッション管理・ORM等の機能が充実している。企業での採用実績も多く、保守性が高い
- **Vue.js**: 学習コストが低く、Laravelとの親和性が高い。Inertia.jsを使用することでSPAのようなUXを実現しつつ、サーバーサイドルーティングを維持できる
- **Docker**: 環境構築の再現性を確保し、本番環境と開発環境の差異を最小化できる

---

#### Q: なぜSPA（シングルページアプリケーション）ではなく、Inertia.jsを使用したのですか？

**A:** 以下のメリットがあるためです：

| 項目 | Inertia.js | 完全なSPA |
|------|-----------|-----------|
| ルーティング | サーバーサイドで管理 | クライアントで管理 |
| 認証 | Laravelの認証をそのまま使用 | API認証が別途必要 |
| SEO | サーバーサイドレンダリング可能 | 追加対応が必要 |
| 開発効率 | 高い（1つのコードベース） | API設計が必要 |

試験システムはSEOが不要で、認証が重要なため、Inertia.jsが最適でした。

---

#### Q: データベース設計で工夫した点は何ですか？

**A:** 以下の点を工夫しました：

- **イベント単位の管理**: 試験会（イベント）ごとに問題セットや設定をカスタマイズ可能
- **柔軟な問題出題**: ランダム/シーケンシャル/カスタムの3モードに対応
- **回答のトラッキング**: ユーザーごとの回答履歴を保持し、分析に活用可能
- **セッション管理**: UUIDベースのセッションで不正防止と再開機能を実現

---

#### Q: このシステムを開発する上で最も苦労した点は何ですか？

**A:** 以下の点で苦労しました：

1. **試験中のセキュリティ対策**: ブラウザのタブ切り替え検知、セッション管理、不正行為の記録など
2. **リアルタイムの制限時間管理**: サーバーとクライアント間での時間同期
3. **Dockerによるカプセル化**: 他の環境でも確実に動作するよう、依存関係を完全に固定

---

#### Q: テストはどのように行いましたか？

**A:** 以下のテスト手法を使用しました：

- **E2Eテスト**: Playwrightを使用した自動化テスト
- **手動テスト**: 実際の試験フローを複数ブラウザで検証
- **負荷テスト**: 複数同時接続時の動作確認

---

### 機能・仕様に関する質問

#### Q: 同時に何人まで受験できますか？

**A:** サーバーのスペックに依存しますが、推奨構成（4コア/8GBメモリ）で50〜100名程度の同時受験を想定しています。大規模な試験の場合はサーバースペックの増強が必要です。

---

#### Q: 試験問題はどのように管理されていますか？

**A:** 管理画面から以下の操作が可能です：

- 問題の新規作成・編集・削除
- パート（第1部〜第3部）ごとの分類
- 選択肢の設定（テキスト・画像対応）
- 正解と解説の設定
- イベントごとの問題割り当て

---

#### Q: 不正行為をどのように防止していますか？

**A:** 以下の対策を実装しています：

| 対策 | 内容 |
|------|------|
| タブ切り替え検知 | ブラウザのフォーカス喪失を記録 |
| セキュリティログ | 違反行為をタイムスタンプ付きで保存 |
| セッション管理 | 1ユーザー1セッションの制約 |
| 自動終了 | 制限時間超過時の自動提出 |

---

#### Q: オフラインでも動作しますか？

**A:** いいえ、このシステムはWebベースのため、常時インターネット（またはイントラネット）接続が必要です。ただし、Dockerコンテナとして動作するため、ローカルネットワーク内での運用も可能です。

---

#### Q: 採点基準をカスタマイズできますか？

**A:** 現在の仕様では、正解1問につき1点の配点です。ランク判定の閾値は問題数に応じて自動的にスケーリングされます。将来的に問題ごとの配点設定機能を追加することも可能です。

---

### 運用・拡張に関する質問

#### Q: システムの保守・運用に必要なスキルは何ですか？

**A:** 以下のスキルが必要です：

| レベル | 必要スキル |
|--------|-----------|
| 基本運用 | Dockerの基本操作（起動・停止・ログ確認） |
| 問題管理 | Webブラウザの基本操作 |
| 障害対応 | Linux基本コマンド、Docker Composeの知識 |
| カスタマイズ | PHP/Laravel、Vue.js/JavaScript の知識 |

---

#### Q: 将来的に機能追加は可能ですか？

**A:** はい、以下のような拡張が考えられます：

- 記述式問題への対応
- 問題ごとの配点設定
- 多言語対応
- 外部システムとのAPI連携
- 詳細な統計・分析機能の強化

---

#### Q: 他社の類似システムとの違いは何ですか？

**A:** 本システムの特徴は以下の通りです：

- **オンプレミス対応**: 社内サーバーで完結するため、データが外部に出ない
- **カスタマイズ性**: オープンソースで提供されるため、自由に改変可能
- **コスト**: SaaS型サービスと異なり、ランニングコストが低い
- **Docker対応**: 環境構築が容易で、ポータブル

---

#### Q: バックアップはどのように取りますか？

**A:** 以下のコマンドでデータベースのバックアップが可能です：

```bash
# バックアップ
docker exec prog-test-prod-db mysqldump -uroot -prootpassword laravel_prod > backup.sql

# リストア
cat backup.sql | docker exec -i prog-test-prod-db mysql -uroot -prootpassword laravel_prod
```

定期的なバックアップはcronジョブで自動化することを推奨します。

---

### 技術的な詳細質問

#### Q: なぜRedisを使用しているのですか？

**A:** 以下の目的でRedisを使用しています：

- **セッション管理**: 高速なセッションデータの読み書き
- **キャッシュ**: データベースへの負荷軽減
- **試験中の一時データ**: 回答の一時保存（制限時間内の回答を保持）

---

#### Q: セキュリティ対策について詳しく教えてください

**A:** 多層防御を採用しています：

```
Layer 1: Cloudflare (SSL/TLS, DDoS防御, WAF)
    ↓
Layer 2: Nginx (リバースプロキシ, レート制限)
    ↓
Layer 3: Laravel (CSRF, XSS対策, SQLインジェクション対策)
    ↓
Layer 4: Database (パスワードハッシュ化, アクセス権限管理)
```

---

#### Q: 本番環境と開発環境はどのように分離していますか？

**A:** Docker Composeファイルを分離しています：

- `docker-compose.yml`: 開発環境
- `docker-compose.prod-test.yml`: 本番テスト環境
- `docker-compose.production.yml`: 本番環境

環境変数は`.env`ファイルで管理し、機密情報はGit管理外としています。

## 2. システム概要

### Q2-1: このシステムは何ですか？

**A:** プログラマー適性試験システムは、プログラマーとしての適性を測定するためのWebベースの試験システムです。3つのパートから構成される試験を実施し、自動採点・ランク判定を行います。

---

### Q2-2: どのような技術で構築されていますか？

**A:** 以下の技術スタックで構築されています：

| レイヤー       | 技術         | バージョン |
| -------------- | ------------ | ---------- |
| フロントエンド | Vue.js       | 3.x        |
| フロントエンド | Tailwind CSS | 3.x        |
| SSRブリッジ    | Inertia.js   | 1.x        |
| バックエンド   | Laravel      | 11.x       |
| データベース   | MySQL        | 8.0        |
| キャッシュ     | Redis        | 7.0        |
| Webサーバー    | Nginx        | Latest     |
| コンテナ       | Docker       | Latest     |

---

### Q2-3: 対応しているブラウザは何ですか？

**A:** 以下のブラウザの最新版で動作確認されています：

- Google Chrome（推奨）
- Mozilla Firefox
- Microsoft Edge
- Safari

※ Internet Explorer は非対応です。

---

### Q2-4: スマートフォンやタブレットで受験できますか？

**A:** PCでの受験を推奨しています。スマートフォンやタブレットでも表示は可能ですが、試験画面のレイアウトはPCに最適化されています。

## 3. 管理者向け

### Q3-1: 初期管理者アカウントは何ですか？

**A:** 初期管理者アカウントは以下の通りです：

| 項目       | 値                |
| ---------- | ----------------- |
| メール     | admin@provisional |
| パスワード | P@ssw0rd          |

⚠️ **重要**: 初回ログイン後、必ずパスワードとメールアドレスを変更してください。

---

### Q3-2: 管理画面にアクセスするURLは？

**A:** `http://[ホスト名]/admin/login` にアクセスしてください。

---

### Q3-3: イベント（試験会）はどのように作成しますか？

**A:** 以下の手順で作成します：

1. 管理者ダッシュボードから「イベント管理」をクリック
2. 「新規イベント作成」ボタンをクリック
3. イベント名、パスフレーズ、開始日時、終了日時を入力
4. 「作成」ボタンをクリック

---

### Q3-4: パスフレーズ（合言葉）とは何ですか？

**A:** 受験者が試験を開始する際に入力するコードです。イベントごとに一意のパスフレーズを設定し、受験者に共有します。これにより、正しいイベントに受験者を紐付けることができます。

---

### Q3-5: 問題の出題方法を変更できますか？

**A:** はい、以下の3つのモードから選択できます：

| モード       | 説明                                           |
| ------------ | ---------------------------------------------- |
| シーケンシャル | 問題番号順に出題（デフォルト）               |
| ランダム     | 問題をランダムな順序で出題                     |
| カスタム     | イベントごとに出題する問題と順序を個別に設定 |

---

### Q3-6: 各パートの問題数や制限時間を変更できますか？

**A:** はい、イベント作成・編集時に以下の項目をカスタマイズできます：

- 第1部の問題数と制限時間
- 第2部の問題数と制限時間
- 第3部の問題数と制限時間

---

### Q3-7: 受験結果はどこで確認できますか？

**A:** 管理者ダッシュボードの「成績管理」から確認できます。以下の情報が閲覧可能です：

- 個別の受験者の詳細な回答内容と得点
- イベント別の統計情報
- ランク別の分布グラフ

---

### Q3-8: 管理者アカウントを追加できますか？

**A:** はい、管理者ダッシュボードの「アカウント管理」から新しい管理者を追加できます。

---

## 4. 受験者向け

### Q4-1: 試験を開始するにはどうすればよいですか？

**A:** 以下の手順で開始します：

1. ブラウザで試験システムのURLにアクセス
2. 試験監督者から共有されたパスフレーズ（合言葉）を入力
3. 氏名・学校名などの情報を入力
4. 練習問題を実施（推奨）
5. 「試験開始」をクリック

---

### Q4-2: 練習問題は必須ですか？

**A:** 練習問題は任意ですが、本番試験の形式に慣れるために受験することを推奨します。練習問題の回答は採点対象外です。

---

### Q4-3: 試験中に画面を離れても大丈夫ですか？

**A:** 試験中にブラウザを閉じたり、他のタブに移動したりすると、セキュリティ違反として記録される場合があります。試験が終了するまでは試験画面にとどまってください。

---

### Q4-4: 回答を変更できますか？

**A:** はい、試験終了前であれば回答を変更できます。回答状況表の問題番号をクリックして、任意の問題に戻ることができます。

---

### Q4-5: 制限時間がなくなったらどうなりますか？

**A:** 各パートには制限時間が設定されています。制限時間が終了すると、自動的にそのパートが終了し、次のパートに進みます。未回答の問題は不正解として扱われます。

---

### Q4-6: 試験結果はいつ確認できますか？

**A:** 試験終了後、自動的に結果画面が表示されます。各パートの得点、総合得点、ランクを即座に確認できます。

---

### Q4-7: 認定証は発行されますか？

**A:** はい、試験終了後に認定証を表示・印刷することができます。

---

## 5. 試験内容

### Q5-1: 試験は何部構成ですか？

**A:** 試験は3部構成です：

| パート | 内容       |
| ------ | ---------- |
| 第1部  | 基礎問題   |
| 第2部  | 応用問題   |
| 第3部  | 実践問題   |

---

### Q5-2: 各パートの画面レイアウトは異なりますか？

**A:** はい、以下のように異なります：

- **第1部・第2部**: 横並びレイアウト（左：問題文と選択肢、右上：回答状況表、右下：ナビゲーション）
- **第3部**: 縦並びレイアウト（上：問題文と選択肢、下：回答状況表とナビゲーション）

---

### Q5-3: ランク判定の基準は何ですか？

**A:** 総合得点に基づいて以下のランクに判定されます：

| ランク   | 名称     | 基準（95問満点時）        |
| -------- | -------- | ------------------------- |
| A        | Platinum | 61点以上                  |
| B        | Gold     | 51点以上61点未満          |
| C        | Silver   | 36点以上51点未満          |
| D        | Bronze   | 36点未満                  |

※ 問題数がカスタマイズされている場合は、基準が問題数に応じてスケーリングされます。

---

### Q5-4: 問題に画像は含まれますか？

**A:** はい、問題文および選択肢には画像を含めることができます。

---

## 6. トラブルシューティング

### Q6-1: ログインできません

**A:** 以下を確認してください：

1. メールアドレスとパスワードが正しいか確認
2. Caps Lockがオフになっているか確認
3. ブラウザのキャッシュとCookieをクリア
4. 別のブラウザで試す

---

### Q6-2: 試験中にセッションが切れました

**A:** 以下の原因が考えられます：

- 長時間操作がなかった
- ネットワーク接続が不安定
- Redisキャッシュの問題

管理者に連絡し、状況を報告してください。

---

### Q6-3: ページが表示されません（502エラー）

**A:** サーバー側の問題の可能性があります。管理者に連絡し、Dockerコンテナの状態を確認してもらってください。

```bash
docker ps  # コンテナ状態確認
```

---

### Q6-4: 設定変更が反映されません

**A:** キャッシュが原因の可能性があります。管理者に以下のコマンドの実行を依頼してください：

```bash
docker exec prog-test-prod-app php artisan config:clear
docker exec prog-test-prod-app php artisan cache:clear
docker exec prog-test-prod-app php artisan view:clear
docker exec prog-test-prod-app php artisan config:cache
```

---

### Q6-5: データベース接続エラーが発生します

**A:** 以下を確認してください：

1. データベースコンテナが起動しているか
2. `.env`ファイルのデータベース設定が正しいか
3. ネットワーク接続に問題がないか

---

## 7. セキュリティ

### Q7-1: データはどのように保護されていますか？

**A:** 以下の多層セキュリティで保護されています：

| レイヤー   | 対策                                  |
| ---------- | ------------------------------------- |
| 通信       | SSL/TLS暗号化（Cloudflare経由時）     |
| アプリ     | CSRF保護、XSS対策、SQLインジェクション対策 |
| 認証       | Laravel Sanctum による認証管理       |
| セッション | Redis によるセッション管理            |
| パスワード | bcrypt によるハッシュ化               |

---

### Q7-2: 試験中の不正行為は検出されますか？

**A:** はい、以下の行為がセキュリティログとして記録されます：

- ブラウザタブの切り替え
- ウィンドウのフォーカス喪失
- 長時間の無操作

記録された違反は管理者が確認できます。

---

### Q7-3: 個人情報はどのように扱われますか？

**A:** 受験者の個人情報（氏名、学校名等）はデータベースに保存されます。管理者のみがアクセス可能で、システム外への送信は行われません。

---

### Q7-4: パスワードポリシーはありますか？

**A:** 管理者アカウントのパスワードは、強力なパスワードを設定することを推奨します。初期パスワードは必ず変更してください。
